# Data Sources

이 문서는 Exportra에서 활용할 공공데이터의 역할, MVP 연동 우선순위, AI 리포트 활용 방식을 정리합니다.

README.md는 범용 오픈소스 프로젝트 소개를 위해 핵심 구조만 설명하고, 세부 데이터 계획은 이 문서에서 관리합니다.

## 1. Purpose

Exportra는 공공 무역·산업 데이터와 AI를 결합하여 제조기업의 수출 의사결정을 돕습니다.

사용자가 제품과 기업 정보를 입력하면 Exportra는 다음 질문에 답하는 것을 목표로 합니다.

- 이 제품에 적합한 HS Code 후보는 무엇인가?
- 어떤 국가가 유망 수출국 후보인가?
- 추천 국가의 시장성, 성장성, 리스크는 어떤가?
- 인증, 규제, 지원사업 측면에서 무엇을 준비해야 하는가?
- 앞으로 3개월 동안 어떤 실행 순서로 움직이면 좋은가?

## 2. Data Usage Principles

- 데이터 출처와 AI 생성 설명을 구분합니다.
- HS Code는 확정값이 아니라 후보로 제시합니다.
- 인증, 통관, 규제 정보는 참고용이며 최종 확인이 필요합니다.
- 검증되지 않은 수치를 사실처럼 단정하지 않습니다.
- MVP에서는 모든 데이터를 실시간 연동하지 않고, 핵심 데이터부터 우선 연동합니다.
- 샘플 데이터는 실제 출처와 수집 기준을 문서화한 뒤 사용합니다.

## 4. Core Data

Core Data는 추천 점수 계산에 직접 활용하는 데이터입니다.

초기 추천 점수는 다음 요소를 조합하는 방향으로 설계합니다.

- 시장 규모
- 최근 성장성
- 지역 수출 연관성
- 국가 리스크
- 지원사업 연계 가능성

점수 산식은 MVP 개발 과정에서 단순하고 설명 가능한 방식으로 시작합니다.

## 7. MVP Integration Plan

초기 MVP에서는 모든 데이터를 한 번에 실시간 연동하지 않습니다. 시연 가능한 핵심 분석 흐름을 먼저 완성하고, 데이터 연동 범위를 단계적으로 넓힙니다.

| 우선순위 | 데이터 | 처리 방식 | 이유 |
| --- | --- | --- | --- |
| 1 | 관세청 국가별 수출입실적 API | API 연동 우선 | 유망 수출국 추천의 핵심 근거 |
| 2 | 한국무역보험공사 국가신용등급 데이터 | 파일 기반 또는 정적 데이터 반영 | 시장성뿐 아니라 국가 리스크를 함께 반영 |
| 3 | KOTRA 국가정보 데이터 | 파일 기반 또는 샘플 데이터 반영 | 추천 국가에 대한 AI 설명 품질 향상 |
| 4 | 관세청 시도별 성질별 수출입실적 API | 샘플 기반 후 연동 | 지역 수출 흐름 설명 강화 |
| 5 | 기업마당 지원사업 API | 샘플 공고 기반 후 연동 | 로드맵과 지원사업 추천 강화 |

## 9. AI Usage Guidelines

AI는 데이터를 해석하고 설명하는 보조 역할을 맡습니다.

AI가 할 수 있는 일:

- 제품 설명을 기반으로 품목 카테고리 정리
- HS Code 후보 추천
- 국가별 수출 규모, 성장성, 리스크를 사람이 읽기 쉬운 문장으로 설명
- 추천 국가별 시장 진입 이유 생성
- 인증, 지원사업, 실행 로드맵을 기업 상황에 맞게 요약

AI가 해서는 안 되는 일:

- HS Code를 확정값처럼 표현
- 인증, 통관, 규제 정보를 법적 자문처럼 단정
- 출처 없는 숫자를 사실처럼 생성
- 공공데이터에 없는 내용을 데이터 기반 사실처럼 표현
