# PRD.md

이 문서는 Exportra의 초기 MVP 범위와 제품 요구사항을 정리합니다.

## 1. 제품 개요

Exportra는 공공 무역·산업 데이터와 AI를 활용하여 제조기업의 해외시장 진출 의사결정을 돕는 AI Export Copilot입니다.

사용자는 기업과 제품 정보를 입력하고, Exportra는 HS Code 후보, 유망 수출국 TOP3, 국가별 시장성·리스크·인증·지원사업·실행 로드맵을 구조화하여 제공합니다.

## 2. 문제 정의

중소 제조기업은 좋은 제품을 보유하고 있어도 해외시장 진출 과정에서 다음과 같은 어려움을 겪습니다.

- 어떤 국가가 유망한지 판단하기 어렵습니다.
- 제품에 맞는 HS Code와 수출입 통계를 해석하기 어렵습니다.
- 국가별 시장성, 경쟁 강도, 리스크를 비교하기 어렵습니다.
- 인증, 규제, 통관 조건을 어디서 확인해야 하는지 알기 어렵습니다.
- 정부 지원사업이나 수출지원 제도를 찾고 연결하기 어렵습니다.
- 무역 전문 컨설팅을 받기에는 시간과 비용 부담이 큽니다.

Exportra는 이 문제를 공공데이터와 AI 기반 설명으로 줄이는 것을 목표로 합니다.

## 3. 핵심 사용자

MVP의 핵심 사용자는 제품을 보유한 제조기업 담당자입니다.

주요 사용자 유형은 다음과 같습니다.

- 해외 진출을 준비하는 중소 제조기업 담당자
- 수출 경험이 부족한 창업기업 또는 소규모 제조기업
- 지역 제조기업
- 기업지원기관 또는 지자체 담당자

## 4. MVP 목표

초기 MVP의 목표는 사용자가 제품과 기업 정보를 입력하면 수출시장 검토에 필요한 핵심 정보를 한 화면에서 확인할 수 있는 시연 가능한 제품을 만드는 것입니다.

MVP는 완성형 상용 서비스가 아니라, 다음 가치를 검증하는 프로토타입입니다.

- 공공데이터 기반 수출시장 추천이 사용자의 의사결정에 도움이 되는가
- AI 설명이 복잡한 무역·시장 데이터를 이해하기 쉽게 만드는가
- 초보 팀이 유지보수할 수 있는 단순한 구조로 구현 가능한가

## 5. 사용자 흐름

1. 사용자가 기업 및 제품 정보를 입력합니다.
2. 사용자가 분석 요청 버튼을 클릭합니다.
3. 시스템이 입력 정보를 바탕으로 제품 특성을 요약합니다.
4. 시스템이 HS Code 후보를 제시합니다.
5. 시스템이 유망 수출국 TOP3를 추천합니다.
6. 시스템이 국가별 분석 카드와 3개월 실행 로드맵을 보여줍니다.
7. 사용자는 데이터 출처와 AI 생성 설명을 함께 확인합니다.

## 6. 입력 정보

MVP에서 받을 입력 정보는 다음과 같습니다.

- 기업명
- 소재 지역
- 제품명
- 제품 설명
- 주요 특징
- 예상 고객
- 수출 경험 여부
- 희망 진출 지역
- 제품 카테고리
- 인증 보유 여부

입력 항목은 MVP 개발 중 화면 복잡도를 고려하여 일부 선택 항목으로 조정할 수 있습니다.

## 7. 출력 정보

MVP에서 제공할 출력 정보는 다음과 같습니다.

- 제품 요약
- HS Code 후보
- 유망 수출국 TOP3
- 국가별 추천 이유
- 시장성 요약
- 경쟁 및 리스크 요약
- 인증 및 규제 체크포인트
- 지원사업 추천
- 3개월 실행 로드맵
- 활용한 데이터 출처

HS Code, 인증, 규제, 통관 정보는 참고용으로 제공하며 최종 확인이 필요합니다.

## 8. MVP 포함 기능

초기 MVP에 포함할 기능은 다음과 같습니다.

- 제품/기업 정보 입력 화면
- 분석 요청 API
- OpenAI API 기반 분석 결과 생성
- HS Code 후보 출력
- 유망 수출국 TOP3 출력
- 국가별 분석 카드 출력
- 인증, 리스크, 지원사업, 로드맵 출력
- 공공데이터 활용 근거 표시
- 결과 화면 유지
- 기본 실행 방법 문서화

## 9. MVP 제외 기능

다음 기능은 MVP 단계에서 제외합니다.

- 회원가입/로그인
- 결제

- 복잡한 권한 관리

- 모바일 앱

- 실제 지원사업 신청 연동

## 10. 데이터 활용 범위

MVP에서는 모든 데이터 소스를 한 번에 실시간 연동하지 않습니다.

우선 활용 데이터는 다음과 같습니다.

| 우선순위 | 데이터 | MVP 처리 방식 | 목적 |
| --- | --- | --- | --- |
| 1 | 관세청 국가별 수출입실적 API | API 연동 우선 | HS Code 기준 국가별 시장 규모와 성장성 분석 |
| 2 | 한국무역보험공사 국가신용등급 데이터 | 파일 기반 또는 정적 데이터 반영 | 국가별 수출 리스크 반영 |
| 3 | KOTRA 국가정보 데이터 | 파일 기반 또는 샘플 데이터 반영 | 국가별 시장환경과 진출 여건 설명 |

후순위 또는 샘플 기반 데이터는 다음과 같습니다.

- 관세청 시도별 성질별 수출입실적 API
- KOTRA 해외인증정보 API
- 기업마당 지원사업 API
- KOTRA 수출바우처사업 모집공고
- 공장등록현황 통계정보

자세한 데이터 활용 계획은 `docs/product/DATA_SOURCES.md`에서 관리합니다.

## 11. AI 역할과 제한사항

AI는 최종 의사결정을 대신하지 않고, 사용자가 공공데이터를 쉽게 이해하고 비교할 수 있도록 돕는 보조 역할을 합니다.

AI가 담당하는 역할은 다음과 같습니다.

- 제품 설명 기반 제품 카테고리 정리
- HS Code 후보 추천
- 국가별 시장 규모와 성장성 요약
- 국가별 리스크와 주의사항 요약
- 인증 및 규제 체크포인트 정리
- 지원사업 추천 이유 생성
- 3개월 실행 로드맵 생성

AI 사용 제한사항은 다음과 같습니다.

- HS Code를 확정값처럼 표현하지 않습니다.
- 인증, 통관, 규제 정보를 법률 또는 무역 자문처럼 단정하지 않습니다.
- 출처 없는 수치를 사실처럼 생성하지 않습니다.
- 공공데이터 기반 정보와 AI 생성 설명을 구분합니다.

## 12. 성공 기준

MVP는 다음 조건을 만족하면 성공으로 봅니다.

- 사용자가 제품 정보를 입력하고 분석 결과를 확인할 수 있습니다.
- 유망 수출국 TOP3가 표시됩니다.
- 각 추천 국가에 대한 이유와 리스크가 표시됩니다.
- HS Code가 확정값이 아닌 후보로 표시됩니다.
- 데이터 출처와 AI 생성 설명이 구분됩니다.
- 팀원이 README를 보고 로컬 실행 흐름을 이해할 수 있습니다.
- 핵심 흐름이 과도한 설정 없이 로컬에서 재현 가능합니다.

## 13. 비기능 요구사항

MVP는 다음 비기능 요구사항을 지향합니다.

- 초보 팀원이 이해하기 쉬운 구조
- 로컬 실행 우선
- API 키와 민감정보를 Git에 커밋하지 않는 구조
- 공공데이터 출처를 확인할 수 있는 결과 화면
- 추후 프론트엔드와 백엔드를 분리 확장할 수 있는 구조
- 복잡한 인프라 없이 시작할 수 있는 구조

## 14. 개발 우선순위

개발은 다음 순서로 진행합니다.

1. 문서와 환경 설정 정리
2. 제품/기업 정보 입력 화면 구현
3. 분석 요청 API 구현
4. 샘플 데이터 기반 추천 로직 구현
5. OpenAI API 연동
6. 결과 화면 구현
7. 데이터 출처 표시
8. README 실행 방법 보완
